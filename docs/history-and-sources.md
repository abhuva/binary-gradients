# Historical And Research Notes

## Core Project Definition

The project explores a specific procedural image pipeline:

```text
1. Generate gradient/scalar field A over the whole canvas.
2. Generate gradient/scalar field B over the whole canvas.
3. Quantize both fields to integers in a power-of-two range.
4. Combine A and B with binary/logical/arithmetic operations.
5. Use the resulting integer as an index into a LUT.
6. Animate by changing gradient parameters and/or cycling the LUT index offset.
```

The important part is not a single `x XOR y` texture, but the generalization to arbitrary integer gradient fields: linear, radial, fan, diamond, box, plasma, noise, etc.

## Origin Story

Marc independently discovered the visual family around 1996 as a teenager, using Turbo Pascal and VGA-era graphics. The initial discovery came from experimenting with line drawing and logical operations such as XOR / AND / OR. Later reasoning clarified that the line drawing itself is not essential; the more general mechanism is integer field composition.

Old photographs exist showing the effect being presented at a science fair. The goal of the repo is not a rights/priority claim, but to preserve and explore the idea properly.

## Closest Known Concepts

### XOR Texture

The classic `x ^ y` texture is well known in demoscene and graphics programming.

Source:
- Lode Vandevenne, "The XOR Texture"
- https://lodev.org/cgtutor/xortexture.html

Relevant points:
- Generated by XORing the pixel coordinates.
- Related AND/OR textures are also discussed.
- Larger-than-256 textures do not automatically add visual quality in 8-bit color unless the value range or color depth is expanded.

### Demoscene / Sizecoding XOR Scrollers

XOR texture scrollers appear in small intro examples.

Source:
- "HOWTO: 4k intros in GNU/Linux"
- https://twiren.kapsi.fi/linux4k.html

Relevant snippet concept:

```c
v[y * w + x] = (x+i) ^ y;
```

### Krita Binary Blending Modes

Krita explicitly documents binary blending modes such as AND, OR, XOR, NAND, NOR, XNOR.

Source:
- https://docs.krita.org/en/reference_manual/blending_modes/binary.html

Important quote summary:
- Binary modes use binary operators for calculations.
- They can generate abstract art from layers with very smooth surfaces.
- They have a fractal attribute.
- The docs show gradient examples for these modes.

This is one of the closest known descriptions to `two smooth gradients + binary operator`.

### XOR Blending With Gradient Layers

A Fandom wiki page describes `XOR Blending` as an art technique using two or three gradient layers and XOR blend mode.

Source:
- https://iaa.fandom.com/wiki/XOR_Blending

Relevant points:
- Uses two or three layers, each containing gradients.
- Different gradient combinations create radically different designs.
- Radial, diamond, and linear gradients are mentioned as useful.
- Gradients become blocky/discrete under XOR blending.

This is very close to the artistic side of this project, but narrower because it focuses on XOR layer blending rather than a generalized procedural field compositor.

### CorelDRAW Logical Merge Modes

CorelDRAW documentation lists logical merge modes:

```text
Logical AND
Logical OR
Logical XOR
```

Source:
- https://community.coreldraw.com/cfs-file/__key/telligent-evolution-components-attachments/00-542-01-00-00-11-13-55/Merge-modes.pdf

Relevant point:
- Merge modes combine source and base colors mathematically.
- Logical modes apply Boolean algebraic formulas to source and base color values.

This likely overlaps with the historical graphics-tool version of the same idea.

### Affinity Forum Discussion

A user discusses trying to recreate old CorelDRAW AND/OR/XOR blend modes, specifically with fullscreen gradient-filled rectangles.

Source:
- https://forum.affinity.serif.com/index.php?/topic/177217-possible-to-create-custom-blend-modes/

Relevant points:
- User used gradient-filled rectangles over a whole design area.
- AND mode was used for a particular visual effect.
- Another participant notes that two gradients can produce increasingly small squares.

This is unusually close to Marc's remembered workflow.

### Bitwise/Sierpinski Structures

Bitwise coordinate operations are mathematically tied to Sierpinski-like structures.

Sources:
- https://math.stackexchange.com/questions/1080223/what-do-bitwise-operators-look-like-in-3d
- https://lcamtuf.substack.com/p/sierpinski-triangle-in-my-bitwise

Relevant points:
- `x XOR y` has self-similar structure.
- `x & y` and predicates such as `(x & y) == 0` reveal Sierpinski-like patterns.
- The effect comes from binary positional representation, not recursive drawing.

### Plasma And Palette Cycling

Palette-cycled procedural effects are a classic demoscene/VGA topic.

Sources:
- https://lodev.org/cgtutor/plasma.html
- https://en.wikipedia.org/wiki/Plasma_effect

Relevant points:
- Plasma uses procedural scalar values and color/palette manipulation.
- Palette cycling is historically important and maps naturally to the LUT stage in this project.
