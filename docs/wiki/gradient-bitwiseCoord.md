# Bitwise coordinate fields

This gradient quantizes pixel coordinates and combines `x` and `y` directly with bitwise operations.

```text
x = floor(pixel.x / scale)
y = floor(pixel.y / scale)
value = op(x, y)
```

Unlike the Render > Combine stage, this operation happens inside one gradient field. It is a coordinate generator.

# Basic modes

`XOR`, `AND`, and `OR` expose the binary structure of the coordinate grid.

`XOR` tends to create checker-like interference. `AND` highlights shared active bits. `OR` fills regions where either coordinate has active bits.

`XNOR`, `NAND`, and `NOR` invert those relationships inside the active value range.

# Shift modes

Shift modes combine `x` with `y << Bit Shift`.

```text
shiftedY = (y << Bit Shift) & valueMask
```

This moves lower y bits into higher bit planes. Visually, that changes the scale at which y influences the pattern.

# Popcount and bit isolation

`Popcount` counts how many bits are set. This turns binary complexity into a scalar density field.

`Lowbit` returns the lowest active bit. `Highbit` returns the highest active bit. These modes are useful for exposing scale hierarchy in coordinates.

# Practical use

Bitwise coordinate fields are the most explicitly digital/geometric fields. They are good for testing how bit planes behave before involving a second gradient.
