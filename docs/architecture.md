# Architecture Overview

## High-Level Pipeline

The runtime pipeline is fully GPU-based:

```text
UI state
  -> WebGL uniforms
  -> fragment shader computes gradient A and gradient B
  -> shader combines integer values
  -> shader samples LUT texture
  -> canvas displays final color
```

No CPU renderer is present. This is intentional because high resolutions and larger value ranges are unusable on CPU.

## Main Files

### `index.html`

Defines the static UI skeleton:

- Left tool panel.
- Tabs: `Render`, `Canvas`, `Grad 1`, `Grad 2`, `LUT`.
- Main canvas viewport.
- LUT editor canvas.
- GPU diagnostics output.

Gradient controls are not hardcoded in HTML. They are generated by `src/app.js`.

### `styles.css`

Compact dev-tool styling:

- Monospace font.
- No rounded UI chrome except the LUT smooth-point marker shape.
- Small controls.
- Tabbed panel.
- Pixelated canvas rendering.
- LUT marker styling.

### `src/app.js`

Contains all runtime logic.

Important top-level sections:

- `FIELD_TYPES`: gradient type registry labels.
- `COMBINE_TYPES`: combine operation labels.
- `COMMON_CONTROLS`: controls shared by all gradients.
- `SPECIFIC_CONTROLS`: type-specific control metadata.
- `DEFAULT_GRADIENT`: full gradient parameter shape.
- `state`: app state.
- WebGL shader source in `initWebGl()`.
- LUT system starting around `makeBuiltinLuts()` / `buildLut()`.
- Viewport pan/zoom handlers.
- GPU diagnostics helpers.

## State Model

The central state object includes:

```js
state = {
  width,
  height,
  valueBits,
  valueRange,
  valueMask,
  rendererActual,
  time,
  timeScale,
  timeRunning,
  previewMode,
  gradients: [gradient1, gradient2],
  combine,
  paletteId,
  paletteOffset,
  cycleSeconds,
  paletteRunning
}
```

`valueBits` currently supports 8..13 bits:

```text
8  -> 256
9  -> 512
10 -> 1024
11 -> 2048
12 -> 4096
13 -> 8192
```

The code queries `gl.MAX_TEXTURE_SIZE` and clamps the runtime value depth if needed.

## Gradient Model

Each gradient is a plain object with all possible parameters. Not every gradient type uses every parameter.

Common parameters:

```text
type
scale
offset
offsetSpeed
```

Origin-based parameters:

```text
originX
originY
originAmpX
originAmpY
originSpeedX
originSpeedY
```

Directional/fan parameters:

```text
angle
rotationSpeed
angleMultiplier
```

Plasma/noise parameters:

```text
freq1
freq2
phaseSpeed
warp
seed
octaves
contrast
driftX
driftY
```

Controls are generated from metadata in `COMMON_CONTROLS` and `SPECIFIC_CONTROLS`.

## WebGL Renderer

`initWebGl()` builds a WebGL2 program with:

- Fullscreen triangle vertex shader.
- Fragment shader that computes both gradient fields.
- Integer combine operations.
- LUT texture lookup.

Important uniforms:

```text
u_resolution
u_time
u_paletteOffset
u_valueRange
u_valueMask
u_combine
u_preview
u_type1
u_type2
u_g1[0]
u_g2[0]
u_lut
```

`gradientUniforms()` serializes each gradient into a fixed-size float array for the shader.

## Preview Modes

The active tab controls what the main canvas shows:

```text
Render -> final result
Canvas -> final result
Grad 1 -> gradient 1 only
Grad 2 -> gradient 2 only
LUT -> final result
```

This is implemented by `state.previewMode` and shader uniform `u_preview`.

## LUT System

Editable LUT definitions use:

```js
{
  id,
  length,
  points: [
    { id, index, color, kind }
  ]
}
```

`kind` is:

```text
smooth -> interpolated stop
hard   -> exact point override
```

Exported JSON is normalized to:

```json
{
  "version": 1,
  "id": "name",
  "length": 256,
  "gradientStops": [],
  "pointOverrides": []
}
```

At render time, the editable LUT is built into an RGB array and then scaled into a GPU texture of width `state.valueRange`.

## Palette Cycling

Palette cycling uses seconds per full cycle, not indices per second:

```text
paletteOffset += dt * valueRange / cycleSeconds
```

This keeps animation speed visually comparable when switching from 8-bit to 13-bit value depth.

## Adding A Gradient Type

1. Add label to `FIELD_TYPES`.
2. Add default parameters to `DEFAULT_GRADIENT` if needed.
3. Add controls to `SPECIFIC_CONTROLS`.
4. Add shader branch in `fieldValue()` using the matching numeric type index.
5. If extra parameters are needed, extend `gradientUniforms()` and the shader uniform array size.
6. Run `node --check src/app.js`.
7. Test in the browser with `GPU Diagnostics` if shader compilation fails.

## Adding A Combine Mode

1. Add label to `COMBINE_TYPES`.
2. Add branch to shader `combineValues()` using the matching numeric index.
3. Run `node --check src/app.js`.
4. Test in browser.

## Known Constraints

- WebGL2 is required.
- No CPU fallback exists by design.
- LUT runtime width is limited by `gl.MAX_TEXTURE_SIZE`.
- Value depth is currently restricted to 8..13 bits.
- Shader integer operations assume power-of-two value ranges.
- `FIELD_TYPES` and shader branch order must stay synchronized.
